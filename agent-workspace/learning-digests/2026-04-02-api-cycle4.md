### 1. Google Places API v2 Migration (YELLOW -- high urgency)
**Impact 9 | Effort 6 | Urgency 9**

The legacy Places API endpoints used in this project will be deprecated. Two files need migration:

**`src/app/api/health/route.ts` (line 33)** -- health check uses legacy Place Details:
```
https://maps.googleapis.com/maps/api/place/details/json?place_id=...&fields=name&key=...
```
Replace with:
```
POST https://places.googleapis.com/v1/places/ChIJN1t_tDeuEmsRUsoyG83frY4
Headers: X-Goog-Api-Key: KEY, X-Goog-FieldMask: displayName
```

**`src/app/api/crawl/area/route.ts` (line 18)** -- area crawl uses legacy Nearby Search:
```
https://maps.googleapis.com/maps/api/place/nearbysearch/json?location=...&radius=...&type=restaurant&key=...
```
Replace with:
```
POST https://places.googleapis.com/v1/places:searchNearby
Body: { includedTypes: ["restaurant"], locationRestriction: { circle: { center: { latitude, longitude }, radius: radiusMeters } } }
Header: X-Goog-FieldMask: places.id,places.displayName,places.formattedAddress,places.location
```

Key v2 changes:
- All requests are POST (not GET)
- API key goes in `X-Goog-Api-Key` header (not query param -- eliminates key leakage in logs)
- Field mask required via `X-Goog-FieldMask` header
- `name` field is now `displayName`; `place_id` is now `id` (or extract from `name` field `places/PLACE_ID`)
- Response field `results` becomes `places`
- `vicinity` becomes `formattedAddress`

Sources: [Google Migration Guide](https://developers.google.com/maps/documentation/places/web-service/legacy/migrate-overview), [Nearby Search (New)](https://developers.google.com/maps/documentation/places/web-service/nearby-search), [Migrate Nearby Search](https://developers.google.com/maps/documentation/places/web-service/legacy/migrate-nearby)

### 2. Security Headers via Next.js Middleware (GREEN)
**Impact 8 | Effort 3**

No `src/middleware.ts` exists. Create one to set security headers on all responses:

**File to create: `src/middleware.ts`**
```ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function middleware(request: NextRequest) {
  const response = NextResponse.next();
  response.headers.set("X-Content-Type-Options", "nosniff");
  response.headers.set("X-Frame-Options", "DENY");
  response.headers.set("Referrer-Policy", "strict-origin-when-cross-origin");
  response.headers.set("Permissions-Policy", "camera=(), microphone=(), geolocation=(self)");
  // CSP: restrict to self + known API domains + image sources
  response.headers.set(
    "Content-Security-Policy",
    "default-src 'self'; script-src 'self' 'unsafe-inline' 'unsafe-eval'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; connect-src 'self' https://maps.googleapis.com https://places.googleapis.com https://api.nal.usda.gov https://api.yelp.com; font-src 'self';"
  );
  return response;
}

export const config = { matcher: ["/((?!_next/static|_next/image|favicon.ico).*)"] };
```

Important: per CVE-2025-29927, middleware must NOT be the sole auth boundary. Auth checks already live in route handlers (good), so this is defense-in-depth only.

Sources: [Next.js CSP Guide](https://nextjs.org/docs/pages/guides/content-security-policy), [Next.js Security Best Practices 2026](https://www.authgear.com/post/nextjs-security-best-practices), [CVE-2025-29927](https://projectdiscovery.io/blog/nextjs-middleware-authorization-bypass)

### 4. Cache-Control Headers for Read-Only Endpoints (GREEN)
**Impact 7 | Effort 2**

Several GET endpoints return data that changes infrequently but currently have no caching headers:

| Route | Recommended Cache-Control |
|-------|--------------------------|
| `GET /api/dishes/[id]` | `public, s-maxage=300, stale-while-revalidate=600` (dish data changes rarely) |
| `GET /api/dishes/[id]/photos` | `public, s-maxage=3600, stale-while-revalidate=7200` (photos very stable) |
| `GET /api/restaurants/[id]` | `public, s-maxage=300, stale-while-revalidate=600` |
| `GET /api/restaurants/[id]/menu` | `public, s-maxage=600, stale-while-revalidate=1200` |
| `GET /api/search/suggest` | `public, s-maxage=60, stale-while-revalidate=120` |
| `GET /api/restaurants/[id]/traffic` | `private, no-cache` (real-time data, must not cache) |
| `GET /api/favorites/*` | `private, no-store` (user-specific) |
| `GET /api/users/profile` | `private, no-store` (user-specific) |

Implementation: add `response.headers.set("Cache-Control", "...")` after the `apiSuccess()` call, or extend `apiSuccess` to accept an options object with cache headers.

Sources: [Next.js API Caching](https://dev.to/melvinprince/mastering-nextjs-api-caching-improve-performance-with-middleware-and-headers-176p), [Next.js Caching Guide](https://nextjs.org/docs/app/getting-started/caching-and-revalidating)

### 6. Missing Rate Limiting on Several Routes (GREEN)
**Impact 6 | Effort 2**

These routes have no rate limiting at all:

| Route | Should Use |
|-------|-----------|
| `GET /api/dishes/[id]` | `withRateLimit("read", handler)` |
| `GET /api/dishes/[id]/photos` | `withRateLimit("read", handler)` |
| `GET /api/restaurants/[id]` | `withRateLimit("read", handler)` |
| `GET /api/restaurants/[id]/menu` | `withRateLimit("read", handler)` |
| `GET /api/restaurants/[id]/traffic` | `withRateLimit("read", handler)` |
| `GET /api/restaurants` | `withRateLimit("read", handler)` |
| `GET /api/admin/image-search` | `withRateLimit("crawl", handler)` -- this scrapes Bing, must be rate-limited |

The `withRateLimit` wrapper makes this trivial: change `export async function GET(...)` to `export const GET = withRateLimit("read", async (request) => { ... })`. The `params` context is passed through.

### 7. Auth Cookie Security Hardening (GREEN)
**Impact 6 | Effort 1**

In `src/app/api/auth/login/route.ts` (line 47) and `register/route.ts` (line 42), the Set-Cookie header is:
```
token=${token}; Path=/; HttpOnly; SameSite=Lax; Max-Age=...
```

Missing `Secure` flag. In production, cookies should always use `Secure` to prevent transmission over HTTP:
```ts
const secure = process.env.NODE_ENV === "production" ? "; Secure" : "";
`token=${token}; Path=/; HttpOnly; SameSite=Lax; Max-Age=${7*24*60*60}${secure}`
```
