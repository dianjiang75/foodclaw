| **CRITICAL** | **Summer Rolls with "peanut sauce" in description passes `allergens=peanuts` filter** — ALLERGEN_KEYWORDS has no "peanuts" key, fallback uses plural "peanuts" which doesn't match singular "peanut" in description | Excluded | Shown to nut-allergic user |
| **CRITICAL** | **Pad Thai Goong passes `diet=nut_free` with 0.95 confidence, no warning** — pad thai commonly contains peanuts. Confidence ≥ 0.90 bypasses KNOWN_ALLERGEN_DISHES check entirely | Warning or exclusion | No warning at all |
| MAJOR | `allergens=` parameter path does NOT enforce `ALLERGY_CRITICAL_MIN` (0.85) confidence — low-confidence dishes (0.76–0.83) shown that would be blocked under `diet=` path | Same safety on both paths | allergens= path is weaker |
| MAJOR | `ingredients_parsed: null` on all dishes — no structured ingredient list for safety audit | Parseable ingredients | Raw description only |
| Minor | No UI disclaimer in filter drawer warning that AI allergen detection is not a substitute for restaurant confirmation | Safety disclaimer | None |

### Evaluator Analysis

The evaluator at `src/lib/evaluator/index.ts` has these correct behaviors:
- `ALLERGY_CRITICAL` includes `nut_free` and `gluten_free` — correct
- `ALLERGY_CRITICAL_MIN` is 0.85 — correct
- Null-flag dishes excluded for critical restrictions under `diet=` — correct
- Warnings generated for confidence 0.85–0.89 — correct

But these dangerous gaps:
- High-confidence (≥0.90) dishes with wrong flags pass silently (Nabeyaki Udon, Tiramisu)
- `KNOWN_ALLERGEN_DISHES.gluten_free` is missing "udon", "tiramisu", "cake", "naan" — only has pasta names
- `ALLERGEN_KEYWORDS` missing "peanuts" entry — singular/plural mismatch causes filter bypass
- `allergens=` code path is a completely separate flow with weaker safety checks

### Suggestions
1. **URGENT**: Add "udon", "tiramisu", "cake", "naan", "bread", "croissant", "pizza" to `KNOWN_ALLERGEN_DISHES.gluten_free`
2. **URGENT**: Add "peanuts" → ["peanut", "peanuts"] to `ALLERGEN_KEYWORDS` map
3. **URGENT**: Apply `ALLERGY_CRITICAL_MIN` confidence gate in the `allergens=` code path too
4. Consider raising `KNOWN_ALLERGEN_DISHES` threshold to 0.95 or always showing a warning for known-risky dishes regardless of confidence
5. Add "Always confirm with restaurant" disclaimer to filter drawer UI

### Score: 2/10
**Multiple life-threatening false positives.** Wheat-flour dishes shown as gluten-free. Peanut dishes shown to nut-allergic users. Two different code paths with different safety levels. This is the highest-priority fix area.
