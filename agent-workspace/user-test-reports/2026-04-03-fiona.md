### Critical Issues for a Quality-Focused Foodie

1. **Templated review summaries** -- Every dish gets the same "Popular dish at X. Customers praise the flavors and portion size." text. This is the single biggest quality failure. Real review synthesis should mention specific flavors, textures, standout qualities, or common criticisms unique to each dish.
2. **No macro source diversity** -- All dishes claim `third_party_db` (USDA/MFP). If the app advertises transparency about where nutrition data comes from, having a single source for everything undermines that value proposition.
3. **ingredients_parsed permanently null** -- The field exists in the schema but is never populated. A nutrition-focused user cannot drill into individual ingredient macros or allergens.
4. **Empty VAPID key** -- Push notifications are non-functional. The infrastructure endpoint exists but is not configured.
5. **photo_count mismatch** -- The detail endpoint says `photo_count: 0` while simultaneously returning 1 photo in the `photos` array.
